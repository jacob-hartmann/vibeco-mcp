# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do not create public GitHub issues for security vulnerabilities.**

Instead, please email security concerns directly to:

**<foss@jacobhartmann.me>**

### What to Include

When reporting a vulnerability, please include:

1. A description of the vulnerability
2. Steps to reproduce the issue
3. Potential impact of the vulnerability
4. Any suggested fixes (optional)

### Response Timeline

I aim to respond to security reports within 48 hours. After the initial response, I will:

1. Confirm the vulnerability and its scope
2. Work on a fix
3. Release a patched version
4. Credit the reporter (unless you prefer to remain anonymous)

## Scope

This security policy applies to:

- The `vibeco-mcp` npm package
- The source code in this repository
